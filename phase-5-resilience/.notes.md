# Phase 5: Resilience

## Goal

- Understand what happens when a dependent service is down/slow
- Implement Circuit Breaker pattern (Resilience4j)
- Implement Fallback methods
- Implement Retry mechanism
- Implement Timeouts

## Services in this phase

- eureka-server
- api-gateway
- user-service, product-service, order-service (with Resilience4j added)

## Why this matters

Phase 3 mein humne dekha tha - agar User Service down ho, Order Service
generic 503 error deta tha. Ab hum seekhenge:

1. Circuit Breaker - baar baar fail ho rahi service ko baar baar call na kare
2. Fallback - graceful default response de, crash na ho
3. Retry - temporary glitch ho toh automatically retry kare
4. Timeout - agar service slow hai, zyada der wait na kare

## Phase 5, Lesson 1: Circuit Breaker

>  Real-Life Analogy 
    
Socho tumhare ghar ka electrical circuit breaker (MCB)  agar kisi wire mein short-circuit ho jaye, MCB turant trip ho jata hai aur poori electricity band kar deta hai. Yeh isliye taaki:

1. Aage aur damage na ho (fire na lage)
2. Tumhe pata chal jaye kuch galat hai
3. Jab tak problem fix na ho, MCB on nahi hota (bar bar try nahi karta)

>Software mein Circuit Breaker bilkul yehi kaam karta hai  agar ek service (jaise User        Service) baar baar fail ho rahi hai, Circuit Breaker "trip" ho jata hai aur turant fail bol deta  hai (bina actually call karne ki koshish kiye) - jab tak User Service theek na ho jaye.

> Circuit Breaker Ke 3 States (Bahut Important)

1. CLOSED (Green) -> Normal State

    Sab kuch theek hai, requests normally User Service ko jati hain.

2. OPEN (Red) -> Failure Detected

    Agar failures ek threshold (jaise 50% requests fail ho rahi hain) cross kar jayein, Circuit Breaker "OPEN" ho jata hai  ab woh User Service ko call hi nahi karta, turant ek fallback response de deta hai (jo hum khud define karenge). Yeh User Service ko "breathing space" deta hai recover karne ke liye.

3. HALF-OPEN (Yellow) -> Testing State

    Kuch time baad (jaise 10 seconds), Circuit Breaker thodi si requests actually try karta hai — "chalo dekhte hain User Service theek hui ya nahi." Agar woh requests successful hon, Circuit CLOSED mein wapas chala jata hai (normal). Agar fail hon, wapas OPEN ho jata hai.

> Fallback Kya Hai (Important Concept)

    Jab Circuit OPEN hota hai, humein kuch toh response dena hai client ko (crash nahi hone dena). Isके liye hum ek fallback method likhte hain  jaise:

        "User details temporarily unavailable, please try again later" (we call it graceful degradation).


## Important Lesson - Service Startup Order
Circuit Breaker aur Feign calls sahi kaam karne ke liye,
services ko sahi order mein start karna zaroori hai:
1. Eureka Server (sabse pehle)
2. User Service, Product Service
3. Order Service
4. API Gateway

Galat order se services properly register/communicate nahi kar pate.
